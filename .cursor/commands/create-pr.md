# PR 생성

## 개요
적절한 설명, 라벨, 리뷰어가 포함된 잘 구조화된 풀 리퀘스트를 생성합니다.

## 단계
1. **브랜치 준비**
   - 모든 변경사항이 커밋되었는지 확인
   - 브랜치를 원격으로 푸시
   - 브랜치가 main과 최신 상태인지 확인

2. **PR 설명 작성**
   - 변경사항을 명확하게 요약
   - 컨텍스트와 동기 포함
   - 호환성을 깨뜨리는 변경사항 나열

3. **PR 설정**
   - 설명적인 제목으로 PR 생성
   - 적절한 라벨 추가
   - 리뷰어 지정
   - 관련 이슈 연결

## PR 템플릿
- [ ] 기능 A
- [ ] 버그 수정 B
- [ ] 단위 테스트 통과
- [ ] 수동 테스트 완료
